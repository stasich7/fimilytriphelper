# История комментариев по плану

Этот файл хранит историю комментариев по плану поездки.

Правила:
- комментарии не удаляются после выпуска новой версии;
- каждый комментарий хранит статус: `open`, `addressed`, `rejected`;
- если комментарий учтен, указывается `resolved_in_version`;
- комментарии группируются по `stable_key`, к которому относятся;
- этот файл хранит историю обсуждения рядом с историей версий плана.

## Шаблон записи

```md
### stable_key=activity.example
title=Название блока

1. author=Имя
   created_at=2026-04-23T10:00:00Z
   status=open
   text=Текст комментария.
```
